# SAP B1 Tax Code - Correct Field Mapping

## Issue Discovered

The Tax Type Combination field was using incorrect values. SAP B1 only accepts specific enum values.

## SAP B1 VatGroups API Fields

### Category Field (BoVatCategoryEnum)

**SAP B1 Only Accepts:**
- `"I"` - Input Tax (Purchasing/Buy)
- `"O"` - Output Tax (Sales/Sell)

**Previous Incorrect Values:**
- ❌ "GST"
- ❌ "CST"
- ❌ "IGST"
- ❌ "VAT"
- ❌ "CENVAT+VAT"
- ❌ "DefineNew"

**Error Message:**
```
Invalid item name 'GST' in Enum 'BoVatCategoryEnum'. 
The valid names are: 'bovcInputTax'-'I', 'bovcOutputTax'-'O'
```

## Correct Implementation

### Frontend Dropdown
```javascript
const TAX_TYPE_COMBINATIONS = [
  { value: "", label: "" },
  { value: "O", label: "Output Tax (Sales)" },
  { value: "I", label: "Input Tax (Purchasing)" },
];
```

### API Payload
```javascript
{
  Code: "GST18",
  Name: "GST @ 18%",
  Category: "O",  // "O" for Output Tax or "I" for Input Tax
  VatGroups_Lines: [
    {
      TaxType: "sys_CGST",
      SalesTaxAccount: "123456",
      PurchasingTaxAccount: "234567",
      NonDeductible: 0,
      NonDeductibleAccount: ""
    }
  ]
}
```

## Understanding Category

### Output Tax (O)
- Used for **Sales** transactions
- Tax collected FROM customers
- Examples:
  - Sales invoices
  - AR invoices
  - Delivery notes

### Input Tax (I)
- Used for **Purchasing** transactions
- Tax paid TO vendors
- Examples:
  - Purchase orders
  - AP invoices
  - Goods receipts

## SAP B1 Tax Code Structure

### Complete Field Mapping

| UI Label | API Field | Type | Values |
|----------|-----------|------|--------|
| Code | Code | string | Max 8 chars |
| Description | Name | string | Max 100 chars |
| Tax Rate | Rate | decimal | Auto-calculated |
| Inactive | Inactive | enum | "tYES" / "tNO" |
| Freight | TaxType | enum | "vgt_Freight" / "vgt_Regular" |
| Category | Category | enum | "O" / "I" |

### VatGroups_Lines Fields

| UI Label | API Field | Type | Notes |
|----------|-----------|------|-------|
| Tax Type | TaxType | string | sys_SGST, sys_CGST, etc. |
| Code | Code | string | Auto-generated by SAP B1 |
| Description | Description | string | Auto-generated by SAP B1 |
| Sales Tax Account | SalesTaxAccount | string | G/L Account Code |
| Purchasing Tax Account | PurchasingTaxAccount | string | G/L Account Code |
| Non-Deductible | NonDeductible | decimal | 0-100 |
| Non-Deductible Account | NonDeductibleAccount | string | G/L Account Code |

## User Guidance

### When to Use Output Tax (O)
- Creating tax codes for sales
- Tax will be charged to customers
- Used in sales orders, invoices, deliveries

### When to Use Input Tax (I)
- Creating tax codes for purchases
- Tax will be paid to vendors
- Used in purchase orders, AP invoices, goods receipts

## Migration Notes

### If You Have Existing Tax Codes
If you created tax codes with incorrect Category values:
1. They won't work in SAP B1
2. You need to recreate them with correct values
3. Use "O" for sales tax codes
4. Use "I" for purchase tax codes

### Testing
Always test tax codes in transactions:
1. Create the tax code
2. Try using it in a sales order (if Category = "O")
3. Try using it in a purchase order (if Category = "I")
4. Verify tax calculation is correct

## SAP B1 Documentation Reference

From SAP B1 Service Layer documentation:

**BoVatCategoryEnum:**
- `bovcInputTax` = "I"
- `bovcOutputTax` = "O"

**VatGroupsTypeEnum:**
- `vgt_Regular` = Regular tax
- `vgt_Freight` = Freight tax
- `vgt_DownPayment` = Down payment tax

## Summary

| Field | Correct Values | Incorrect Values |
|-------|---------------|------------------|
| Category | "O", "I" | "GST", "VAT", "CST", etc. |
| TaxType (Freight) | "vgt_Freight", "vgt_Regular" | true/false |
| Inactive | "tYES", "tNO" | true/false |

## Conclusion

The Category field must use SAP B1's enum values:
- ✅ "O" for Output Tax (Sales)
- ✅ "I" for Input Tax (Purchasing)
- ❌ NOT custom values like "GST", "VAT", etc.

This is a fundamental SAP B1 requirement and cannot be changed.
